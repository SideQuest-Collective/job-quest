# Milestone v1.0 Audit

## Summary

The milestone is close, but it does not meet the workflow's definition of done yet. Most of the dual-runtime wiring is in place and the dashboard runtime metadata path is working, but two end-to-end product flows are still broken in practice:

1. Codex daily intel execution does not complete successfully through the installed helper path.
2. The dashboard and the generator disagree on what "today" means because one uses UTC dates and the other uses local dates.

That is enough to force `gaps_found`.

## Cross-Phase Integration

### Working

- Install -> runtime descriptor -> dashboard runtime endpoint is wired correctly.
- Shared-home registration and schedule command hint are exposed correctly through `/api/runtime` and `/api/job-status`.

### Broken

- Runtime execution layer -> scheduled intel -> dashboard daily status:
  The generator writes local-date files, but the dashboard looks up UTC-date files.
- Validation artifacts -> release confidence:
  The milestone was marked complete in planning artifacts before these E2E gaps were surfaced.

## Evidence

### RT-02: Codex daily intel flow fails

- `skill/bin/run-daily-intel.sh:105-119` runs the Codex path and records success/failure.
- `lib/runtime-shell.sh:79-91` shells out to `codex exec` with the prompt file.
- `~/.job-quest/data/logs/daily-intel.log` recorded:
  - run started at `2026-04-23 15:20:25`
  - run failed at `2026-04-23 15:25:31`
  - exit code `1`
  - nested Codex execution entered but did not produce the expected summary/output files

### PROD-01: Dashboard date handling is inconsistent with generated artifacts

- `app/server.js:74`, `87`, `140`, `169`, and `1083` derive "today" from `new Date().toISOString().split('T')[0]`.
- During this audit:
  - `/api/job-status` returned `date: "2026-04-24"` with `intel.ready: false`
  - `/api/intel/latest` served `date: "2026-04-23"`
  - `/api/tasks/today` served `date: "2026-04-23"`
- This happened on `2026-04-23` in `America/New_York`, which is a concrete local/UTC rollover mismatch.

### VER-02: Validation proof is incomplete

- `docs/validation/dual-runtime-checklist.md:63-78` explicitly says helper scripts should be run and that daily intel should write JSON files into `~/.job-quest/data/`.
- The observed Codex helper failure means the checklist was not actually proven green for the full supported flow.
